## 二、产品概述

### 2.1 产品定位
小帮手是一款面向**单个用户**的**私人知识管理助手**。
它帮助用户从日常聊天内容中提取有价值的信息，经过 AI 整理和人工确认后，沉淀为可检索、可复用的个人知识资产。

### 2.2 MVP 核心目标
MVP 阶段聚焦验证以下核心闭环是否成立：

> 聊天内容 → AI 提炼候选知识 → 用户确认沉淀 → 可检索复用

MVP 重点验证两个问题：

1. 用户是否愿意将聊天中的内容沉淀为结构化知识
2. 用户是否会持续使用检索能力回看和复用这些知识

### 2.3 核心价值
- **降低记录成本**：减少手动整理笔记的负担
- **提升沉淀效率**：AI 自动提炼候选知识
- **保障知识质量**：通过人工确认控制沉淀内容
- **提升复用效率**：支持自然语言检索与来源追溯

## 三、目标用户

### 3.1 用户画像
- 个人知识工作者
- 日常聊天和信息输入频繁
- 有知识沉淀需求，但缺少系统整理时间
- 希望历史信息可被持续检索和复用

### 3.2 MVP 阶段目标用户范围
MVP 阶段暂聚焦于：
- 研发
- 产品经理
- 项目协作者

说明：以上用户群具有较高的信息密度、知识复用需求和沉淀价值，适合作为首批验证对象。

## 四、MVP 范围定义

### 4.1 MVP 必做范围

#### 4.1.1 数据输入
MVP 仅支持以下两类输入：
1. 用户与小帮手的私聊内容
2. 用户手动转发 / 手动标记的消息内容

#### 4.1.2 知识整理
系统对输入内容进行 AI 提炼，生成候选知识条目，包含：
- 标题
- 摘要
- 正文内容
- 标签
- 来源
- 置信度

#### 4.1.3 人工确认
用户可对候选知识执行以下操作：
- 沉淀
- 忽略
- 稍后处理

支持基础批量处理。

#### 4.1.4 知识入库
确认沉淀后的知识进入个人知识库，并建立来源映射关系。

#### 4.1.5 基础检索
用户可通过自然语言查询自己的知识库，获得：
- AI 总结答案
- 相关知识条目
- 来源引用链接

#### 4.1.6 飞书文档输出
系统将已沉淀知识同步到飞书文档，便于用户浏览与阅读。

### 4.2 MVP 明确不做范围

<callout background-color="light-yellow">
以下内容不纳入首版，以保证 MVP 范围收敛和交付节奏可控。
</callout>

- 自动监控其他联系人 / 群聊消息
- 周期自动整理
- 微信等多平台输入
- 多人协作编辑
- 对外公开分享
- MCP 对其他 agent 开放
- 复杂权限体系
- 飞书文档反向回写数据库
- 移动端版本
- 高级推荐、信誉分、操作排行等运营能力

## 五、产品边界定义

### 5.1 用户边界
首版仅面向**单用户私有知识管理**，不处理团队共享知识协作场景。

### 5.2 数据边界
首版仅处理以下数据：
- 用户主动与小帮手产生的内容
- 用户主动转发 / 标记进入系统的内容

系统不主动采集未经用户明确送入的数据。

### 5.3 文档边界
MVP 中飞书文档的角色为：

**展示层 / 浏览层**

不是：

**事实源 / 编辑主入口**

明确原则：
- 数据库是唯一事实源
- 飞书文档由系统生成和更新
- 用户在飞书中的手工编辑不回写数据库

## 六、核心用户流程

### 6.1 主流程概述

#### 流程一：记录
用户通过私聊、转发或标记的方式，将内容送入待整理队列。

#### 流程二：整理
系统通过 AI 对待整理内容进行提炼，生成候选知识卡片。

#### 流程三：确认
用户对候选知识进行处理：
- 沉淀
- 忽略
- 稍后处理

#### 流程四：入库
已确认沉淀的知识写入数据库，并建立检索索引与飞书文档映射。

#### 流程五：检索
用户使用自然语言检索知识，系统返回答案、条目和来源。

### 6.2 用户流程示意

```plaintext
用户聊天 / 转发 / 标记消息
        ↓
进入待整理队列
        ↓
AI 生成候选知识
        ↓
用户确认（沉淀 / 忽略 / 稍后处理）
        ↓
写入知识库
        ↓
建立检索索引
        ↓
同步飞书文档
        ↓
用户自然语言检索
```

## 七、功能需求

### 7.1 功能一：记录内容

#### 功能说明
用户可通过与小帮手对话，或主动转发 / 标记消息，将信息送入系统。

#### 验收标准
- [ ] 用户与小帮手的聊天内容可进入系统
- [ ] 用户可手动转发消息给小帮手
- [ ] 用户可标记消息进入待整理队列
- [ ] 系统可记录来源信息

### 7.2 功能二：AI 生成候选知识

#### 功能说明
系统自动从输入中提炼候选知识卡片，供用户后续确认。

#### 候选知识最小结构
- 标题
- 摘要
- 内容
- 标签
- 来源
- 置信度

#### 验收标准
- [ ] 系统可根据输入生成候选知识
- [ ] 候选知识包含完整基础字段
- [ ] 用户可查看候选知识列表
- [ ] 每条候选知识可追溯到来源

### 7.3 功能三：人工确认

#### 功能说明
用户对候选知识进行审核处理，确保入库内容可控。

#### 支持操作
- 沉淀
- 忽略
- 稍后处理

#### 批量能力
支持对多条候选知识进行批量操作。

#### 验收标准
- [ ] 用户可对单条候选知识执行三类处理
- [ ] 用户可批量处理候选知识
- [ ] 操作结果可被明确反馈
- [ ] 稍后处理的内容进入待办区域或待处理列表

### 7.4 功能四：知识入库

#### 功能说明
被确认沉淀的知识写入个人知识库，形成结构化资产。

#### 验收标准
- [ ] 已沉淀知识进入知识库列表
- [ ] 知识条目支持查看详情
- [ ] 知识条目保留来源信息
- [ ] 可记录创建时间与更新时间

### 7.5 功能五：知识检索

#### 功能说明
用户可以通过自然语言查询个人知识库，获取总结和引用。

#### 返回结果
- AI 总结答案
- 相关知识条目
- 来源引用 / 跳转链接

#### 验收标准
- [ ] 支持自然语言查询
- [ ] 返回结果包含答案与引用
- [ ] 用户可跳转查看知识详情
- [ ] 用户可跳转查看来源内容

### 7.6 功能六：飞书文档输出

#### 功能说明
已沉淀知识同步生成到飞书文档，便于用户进行浏览式阅读。

#### 验收标准
- [ ] 已沉淀知识可同步到飞书文档
- [ ] 飞书文档可作为阅读入口
- [ ] 每条知识可关联飞书文档链接
- [ ] 文档内容与知识条目建立映射关系

## 八、知识条目模型

MVP 阶段每条知识至少包含以下字段：

<lark-table header-row="true">
| 字段 | 说明 |
| --- | --- |
| id | 知识唯一标识 |
| title | 标题 |
| summary | 摘要 |
| content | 正文内容 |
| tags | 标签列表 |
| source_type | 来源类型 |
| source_link | 来源链接 |
| source_message_id | 来源消息标识 |
| confidence | AI 置信度 |
| status | 当前状态 |
| created_at | 创建时间 |
| updated_at | 更新时间 |
| doc_link | 飞书文档链接 |
</lark-table>

### 状态建议
- `draft`
- `pending_review`
- `approved`
- `ignored`
- `later`
- `sync_failed`

## 九、页面范围

MVP 页面建议收敛为以下 5 个核心页面：

1. **登录页**：飞书登录、首次进入基础引导
2. **待确认页**：查看候选知识、单条处理、批量处理
3. **知识库页**：查看已沉淀知识列表、基础筛选、进入知识详情
4. **检索页**：自然语言搜索、展示答案、条目、来源
5. **知识详情页**：查看结构化知识内容、查看来源、查看飞书文档链接

## 十、非功能要求

### 10.1 性能要求
- 候选知识列表可稳定加载
- 基础检索响应时间应控制在可接受范围内
- 批量处理 20 条候选知识时体验可接受

### 10.2 可追溯性要求
系统必须支持从一条知识追溯到：
- 来源消息
- 候选草稿
- 审核结果
- 飞书文档映射

### 10.3 一致性要求
- 数据库为唯一事实源
- 检索索引与飞书文档属于派生层
- 派生层失败应支持补偿和重试

### 10.4 幂等要求
- 消息采集要支持重复事件防重
- 文档同步要支持重试不重复写入
- 人工确认操作要避免重复提交带来的异常状态

## 十一、MVP 验收标准

### 11.1 功能验收
- [ ] 用户可将内容送入待整理队列
- [ ] 系统可生成候选知识
- [ ] 用户可执行沉淀 / 忽略 / 稍后处理
- [ ] 已沉淀知识可进入知识库
- [ ] 用户可进行自然语言检索
- [ ] 检索结果包含来源引用
- [ ] 已沉淀知识可同步到飞书文档

### 11.2 体验验收
- [ ] 单条确认操作简单直接
- [ ] 批量处理路径清晰
- [ ] 空状态、加载状态、错误状态可感知
- [ ] 搜索结果可快速跳转到知识详情或来源

### 11.3 技术验收
- [ ] 采集链路支持幂等
- [ ] 入库、索引、文档同步支持重试
- [ ] 检索链路可稳定运行
- [ ] 关键操作具备日志与追踪能力
