## 二、一句话定义

knowledgeBook是一款面向**单个用户**的**飞书机器人驱动的私人知识管理助手**：
用户通过飞书机器人输入内容，knowledgeBook生成候选知识，用户确认并分类沉淀，系统同步到飞书文档，后续用户再通过飞书机器人检索、更新、软删除或恢复知识。

---

## 三、MVP 目标

MVP 只验证一件事：

> 用户是否愿意把飞书聊天中的信息沉淀成可分类、可维护、可检索、可复用的个人知识资产。

围绕这个目标，MVP 重点验证三类行为：
1. 用户是否愿意沉淀知识
2. 用户是否愿意持续检索和复用知识
3. 用户是否愿意持续维护知识（更新、改分类、软删除、恢复）

---

## 四、MVP 范围

## 4.1 保留能力

### 1. 输入
- 用户与knowledgeBook飞书机器人私聊输入
- 用户主动转发聊天记录给knowledgeBook

### 2. 候选知识生成
系统生成：
- 标题
- 摘要
- 正文内容
- 标签
- 推荐分类
- 置信度

### 3. 人工确认
支持：
- 沉淀
- 忽略
- 稍后处理

### 4. 分类体系
- 支持多级分类
- 最大层级暂定 5 级
- 优先遵循用户给定分类标准
- 若无用户标准，系统生成初始一级 / 二级分类并智能推荐
- 高置信分类允许自动采纳

### 5. 知识管理
- 更新知识标题 / 内容 / 分类
- 软删除知识
- 30 天内恢复软删除知识

### 6. 检索
返回：
- AI 总结答案
- 相关知识条目
- 飞书文档对应位置链接

### 7. 飞书文档展示
- 以分类子文档为主
- 子文档嵌套 + 多级标题交替使用
- 文档层级最大暂定 5 级
- 内容过多或过少时自动调整分级方式

### 8. 部署交付
- Docker Compose 部署方式
- 用户部署手册

---

## 4.2 明确不做

- 独立 Web 页面或后台
- 原始聊天追溯
- 群聊 / 联系人自动监控
- 多平台输入（如微信）
- 多人协作
- 分享能力
- MCP 对外开放
- 复杂权限体系
- 飞书文档自动回写数据库

## 五、系统边界

## 5.1 交互载体

| 载体 | 职责 |
| --- | --- |
| 飞书机器人 | 输入、确认、待办、检索、更新、软删除、恢复、文档修改同步回收 |
| 飞书文档 | 分类化展示知识、承载浏览入口、提供文档位置链接 |

## 5.2 数据事实源
- 数据库是唯一事实源
- 飞书文档是派生展示层

## 5.3 文档手改规则
- 用户手动修改飞书文档后，默认**不自动回写数据库**
- 若用户显式发出同步指令，knowledgeBook读取修改部分并同步回数据库
- 若用户不发出同步指令，后续数据库同步文档时会直接覆盖这些手工修改

## 六、核心流程

## 6.1 主链路
```plaintext
飞书机器人输入
→ 生成候选知识
→ 推荐分类
→ 用户确认沉淀
→ 入库并生成版本
→ 同步到飞书文档
→ 用户通过飞书机器人检索
→ 返回答案 + 条目 + 文档位置链接
```

## 6.2 待办链路
```plaintext
候选知识
→ 用户选择“稍后处理”
→ 进入待办列表
→ 后续重新处理为沉淀或忽略
```

## 6.3 维护链路
```plaintext
已沉淀知识
→ 更新 / 调整分类 / 软删除
→ 生成新版本
→ 同步飞书文档
→ 若软删除则进入 30 天恢复窗口
```

## 6.4 文档同步回收链路
```plaintext
用户手动修改飞书文档
→ 用户显式发送同步指令
→ 系统读取修改部分
→ 与数据库当前版本对比
→ 如有差异，写入新版本并重新同步
```

## 七、分类与文档组织规则

## 7.1 分类规则
- 最大层级：5 级
- 优先级：用户标准 > 已有分类树 > AI 推荐
- 系统不鼓励无边界扩张分类树

## 7.2 高置信自动分类
建议默认条件：
- Top1 置信度 ≥ 0.85
- Top1 与 Top2 分差 ≥ 0.15

满足时可自动分类，否则要求人工确认。

## 7.3 飞书文档组织规则
优先采用：
```plaintext
知识库总览文档
├── 一级分类子文档
│   ├── 二级分类子文档
│   │   ├── 三级标题 / 子文档
│   │   │   ├── 四级标题 / 五级标题
│   │   │   │   ├── 知识条目
```

## 7.4 自适应分级
- 内容过多：拆分为下一级子文档
- 内容过少：合并为当前文档的标题层级
- 目标：避免文档过密或过稀

## 八、知识生命周期

## 8.1 候选知识状态
- `PENDING_REVIEW`
- `IGNORED`
- `LATER`
- `APPROVED`

## 8.2 正式知识状态
- `ACTIVE`
- `REMOVED_SOFT`

## 8.3 软删除规则
- 软删除后默认不参与检索
- 数据保留 30 天
- 用户可在 30 天内恢复
- 超过 30 天自动永久清理

## 十一、已确认决策

1. 仅支持飞书机器人输入，也支持转发消息给机器人。  
2. V1.0 不建设独立 Web 页面。  
3. 检索结果仅返回“答案 + 条目 + 飞书文档位置链接”。  
4. 支持稍后处理待办。  
5. 支持知识更新、改分类、软删除、恢复。  
6. 分类优先遵循用户标准；无标准时系统推荐。  
7. 最大层级暂定 5 级。  
8. 飞书文档以分类子文档为主。  
9. 文档手改默认不回写数据库，需显式同步。  
10. 软删除保留 30 天，超期永久清理。  
11. 提供 Docker Compose 部署与用户部署手册。
